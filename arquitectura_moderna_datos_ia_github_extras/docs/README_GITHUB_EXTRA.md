# Guía de uso

Los ejemplos están ordenados por capítulos. Se recomienda avanzar en este orden:

1. `00_data_lab`: generación de datos sintéticos y modelo operativo.
2. `02_ingestion_transformation`: ingesta, raw, staging, dbt y Git.
3. `03_modeling_patterns`: Kimball, Data Vault y patrones híbridos.
4. `04_quality_serving`: calidad, KPIs, serving y semantic layer.
5. `05_governance_incentives`: contratos, gobierno, políticas y metadatos.
6. `06_ai_preparation`: RAG, embeddings, prompts y activos IA.
7. `07_ai_agents`: agentes, tools, guardrails y evaluación.
8. `08_decision_intelligence`: predicción, recomendación, decisión y feedback.
9. `09_enterprise_architecture_gitops`: GitOps, IaC, Terraform y entornos.
10. `10_data_spaces_gaiax`: data products, políticas y espacios de datos.
